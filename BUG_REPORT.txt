BUG #1: NON-UNIQUE ID ATTRIBUTES
================================================================================
Severity: Low
Priority: Low

Description:
Multiple form fields share the same ID attribute (#field-1), violating HTML standards.

Steps to Reproduce:
1. Navigate to https://app.qa.nesto.ca/signup
2. Open browser developer console
3. Observe warning: "Found 7 elements with non-unique id #field-1"

Expected Behavior:
Each form field should have a unique ID attribute.

Impact:
- Accessibility issues for screen readers
- JavaScript selector conflicts
- HTML validation errors

================================================================================

BUG #2: NAME FIELDS ACCEPT SPECIAL CHARACTERS
================================================================================
Severity: Medium
Priority: Medium

Description:
First name and last name fields accept special characters, numbers, and symbols without validation.

Steps to Reproduce:
1. Navigate to https://app.qa.nesto.ca/signup
2. Enter "John@123" in First name field
3. Enter "Doe#Smith" in Last name field
4. Observe that form accepts these values without validation errors

Expected Behavior:
Name fields should:
- Only allow letters, spaces, hyphens (-), and apostrophes (')
- Reject special symbols (@, #, $, %, etc.) and numbers
- Show validation error messages for invalid input

Impact:
- Data quality issues
- Potential security concerns
- Poor user experience

Test Case: TC-005 (Skipped - documents bug behavior)

================================================================================

BUG #3: PHONE NUMBER FIELD ALLOWS UNLIMITED INPUT
================================================================================
Severity: Medium
Priority: Medium

Description:
Phone number field allows unlimited input without country-based validation or length restrictions.

Steps to Reproduce:
1. Navigate to https://app.qa.nesto.ca/signup
2. Enter "123" (too short) in phone number field
3. Observe no validation error
4. Enter 30+ digits in phone number field
5. Observe no validation error

Expected Behavior:
For Canadian phone numbers (default):
- Minimum: 10 digits
- Maximum: 11 digits (with country code)
- Should reject numbers that are too short/long
- Should validate format and area codes

Impact:
- Data quality issues
- Poor user experience
- Potential for incorrect phone numbers in system

Test Case: TC-007 (documents bug behavior)

================================================================================

BUG #4: GENERIC WEAK PASSWORD ERROR MESSAGE
================================================================================
Severity: Medium
Priority: Medium

Description:
When a user submits a weak password, the error message is too generic ("invalid password") and doesn't provide clear guidance on why the password was rejected or how to fix it.

Steps to Reproduce:
1. Navigate to https://app.qa.nesto.ca/signup
2. Fill form with valid data
3. Enter a weak password (e.g., "Password1234")
4. Submit form
5. Observe generic error message: "invalid password"

Expected Behavior:
Error message should:
- Clearly indicate the password is too weak
- Provide specific guidance on password strength requirements
- Explain what makes a password strong (e.g., "Password is too weak. Please use a stronger password with more complexity.")
- Be user-friendly and actionable for new users

Impact:
- Poor user experience - users don't understand why password was rejected
- Confusion for new users unfamiliar with password requirements
- Increased support requests
- Potential user abandonment

Test Case: TC-017 (documents bug behavior)

================================================================================

BUG #5: GENERIC DUPLICATE EMAIL ERROR MESSAGE
================================================================================
Severity: Medium
Priority: Medium

Description:
When a user attempts to signup with an email that already exists, the error message is too generic ("duplicate entity") and doesn't provide clear guidance on what went wrong or how to fix it.

Steps to Reproduce:
1. Navigate to https://app.qa.nesto.ca/signup
2. Fill form with valid data
3. Enter an email that already exists in the system (e.g., "test@example.com")
4. Submit form
5. Observe generic error message.

Expected Behavior:
Error message should:
- Clearly indicate that the email address is already registered
- Provide actionable guidance (e.g., "This email address is already registered. Please use a different email or try logging in.")
- Be user-friendly and understandable for new users
- Suggest alternative actions (login, use different email, reset password)

Impact:
- Poor user experience - users don't understand what "duplicate entity" means
- Confusion for users who may have forgotten they already have an account
- Increased support requests
- Potential user abandonment
- Users may try to create multiple accounts with slight email variations

Test Case: TC-020 (documents bug behavior)
